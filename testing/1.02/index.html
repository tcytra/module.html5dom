1.02

Provide the ability to search an existing dom node tree for matching elements via a constructor argument

Notes

  • The method to find or search will invoke an instance of a new object, ie Html5Search
  • The search object will generate a list of nodes that match the provided constructor
  • The list object will provide access to common methods for node manipulation, ie: append
  • The search should be able to be invoked internal to an Html5 object, or as a standalone instance
  • The Html5Construct will require the ability to suppress assuming a default nodeName, currently <div>
  • Require the ability to exclude results via construct argument; Html5Search::find()->not() method
  • Require the ability to chain find methods to narrow discovered results
