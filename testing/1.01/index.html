Reorganize the Html5 class hierarchy and instance configuration

Notes

  • The Html5 object should be an abstract class, remove the __construct, move the functionality to Html5Document
  • Html5Document should be the only of the existing classes directly extending the Html5 object
  • Html5DocumentHead and Html5DocumentFragment should extend the Html5Document class
  • Consider the option of adding an Html5DocumentBody object; what are the benefits? Prefabricate pages?
  • Consider the option of adding an Html5DocumentNode object, extends Html5, parent to Element and Fragment?
  • Consider the option of adding an Html5Config object to manage the initial configuration as an object
